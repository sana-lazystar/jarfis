# Project Profile Template

아래 구조로 `./jarfis/project-profile.md`를 생성한다.
해당 depth에서 분석하지 않은 섹션은 포함하지 말 것 (빈 섹션 금지).

```markdown
# Project Profile: [프로젝트명]

> Generated by `/jarfis:project-init` on YYYY-MM-DD
> Depth: basic | medium | deep
> Type: BE | FE | Fullstack

---

## Tech Stack
- **Language**: (예: TypeScript 5.x)
- **Runtime**: (예: Node.js 22)
- **Framework**: (예: Fastify 5)
- **Package Manager**: (예: pnpm 9)
- **Key Dependencies**: (핵심 라이브러리 목록)

## Directory Structure
(압축된 트리 — 주요 디렉토리만, 역할 주석 포함)

## Scripts & Commands
| 작업 | 커맨드 |
|------|--------|
| Dev | `pnpm dev` |
| Build | `pnpm build` |
| Test | `pnpm test` |
| Lint | `pnpm lint` |

## Config Summary
(ESLint, TS, 빌드 도구 등의 핵심 설정 — 전체 파일이 아닌 요약)

## Coding Conventions (medium+)
- **파일 네이밍**: (예: kebab-case.ts)
- **컴포넌트 구조**: (예: feature-based co-location)
- **Import 규칙**: (예: @/ 절대경로, barrel exports 사용)
- **타입 정의**: (예: interface 선호, *.types.ts 분리)
- **에러 핸들링**: (예: 커스텀 AppError 클래스 + 글로벌 핸들러)

## API Routes / Pages (medium+)
(엔드포인트 또는 페이지 라우트 목록 — Method, Path, 핸들러/컴포넌트 위치)

## Data Models (medium+)
(모델명 + 주요 필드 요약표)

## Module Map (medium+)
(주요 모듈 간 의존 관계 요약)

## Architecture (deep)
- **패턴**: (예: Controller → Service → Repository 레이어)
- **미들웨어**: (예: auth → validation → handler)
- **상태 관리**: (FE인 경우)
- **인증 방식**: (예: JWT + refresh token rotation)

## Business Logic Summary (deep)
(주요 서비스/모듈별 책임 요약 — 1-2문장씩)

## Reusable Components (deep)
(공통 컴포넌트/유틸리티 카탈로그)

## Notes & Caveats (deep)
(기술적 특이사항, TODO/FIXME 요약, 주의점)
```
