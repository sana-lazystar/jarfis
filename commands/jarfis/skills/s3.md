## Common Pitfalls
- **Public ACL 실수**: Block Public Access 기본 ON(2023+), 하지만 legacy 버킷은 여전히 public 가능. **Bucket Policy 우선 사용**, ACL 지양.
- **Presigned URL 만료**: 기본 15분, 최대 7일(v4 서명). TTL 짧게 + 클라이언트 재발급 로직.
- **Large file 직접 PutObject**: 5GB 제한. 그 이상 또는 중단 복구 필요 → **Multipart Upload** (5MB~5GB/part).
- **CORS 미설정**: 브라우저 upload 실패. `AllowedOrigins`, `AllowedHeaders`, `ExposeHeaders: [ETag]` 필수.
- **List 성능**: prefix 없이 list → O(N). 접두어 설계 중요 (날짜 기반 등).

## Decision Heuristics
- Storage class: 접근 빈도로 결정. Standard → Intelligent-Tiering (자동) → IA → Glacier.
- Lifecycle rule: 30일 후 IA, 90일 후 Glacier, 365일 후 Deep Archive — 보관용은 비용 90%+ 절감.
- Encryption: SSE-S3 (default, AES-256) vs SSE-KMS (감사/키 로테이션). PII는 SSE-KMS + key policy.
- 대량 업로드: CloudFront 엣지 → S3 Transfer Acceleration (2배 빠름, 약간 비쌈).

## Anti-patterns
- **Secret을 S3 object name에 포함**: ACL 정책 실수로 노출 위험. 내부 ID + metadata 분리.
- **Bucket per user**: 100 bucket 하드캡. `prefix/{userId}/` 설계.
- **Server-side processing을 S3 event로 trigger하고 동일 bucket에 쓰기**: 무한 루프. 다른 bucket 또는 prefix 필터.

## Version & Environment Notes
- Object Lock (WORM): 규제 감사용. Governance mode vs Compliance mode (후자는 root도 삭제 불가).
- S3 Express One Zone (2023+): 단일 AZ 고성능, 8배 빠른 latency. 데이터 분석/ML 파이프라인용.
- Cross-region replication: KMS 키는 버전별로 별도 복제 설정.
