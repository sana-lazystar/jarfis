# Phase 3: Design (figma + text unified)

> Executed by **jarfis-foreman** inside tmux. Runs only when `state.design.mode != null`.
> jarfis-foreman is the **orchestrator**: spawns ux-designer (and PO for review rounds) via Task. It does NOT read figma-spec YAML or design HTML itself — those belong to the sub-agent.
> Minimal orchestrator-only work: `cp` of existing token-map baseline (text mode), `jq` on state.
> All sub-agent artifacts in English. User-facing messages in $LOCALE.
> Output contract is identical across modes: `$DOCS_DIR/design/{page_path}/index.html` + `reference*.png`.

**Execution context**:
- `$DOCS_DIR` = tmux workspace
- `$STATE_FILE` = `$DOCS_DIR/.jarfis-state.json`

### jarfis-foreman precompute (from $STATE_FILE)

- `$DESIGN_MODE` = `state.design.mode` ∈ {"figma", "text"}
- `$RESPONSIVE` = `state.responsive` ∈ {"pc-only", "pc-mobile", "pc-mobile-tablet"}
- `$FIGMA_PAGES_JSON` = `state.design.figmaPages` (figma mode; array of `{title, url}`)
- `$ORG_ROOT` = `state.org.root` (empty string when org not registered)
- `$COMMON_SKIP_JSON` = `state.design.common_components_skip` (figma mode; array or `"[]"`; set by main session before Phase 3 — see "Common-component handling" below)

## Required Inputs (consumed by ux-designer sub-agent)

- `$DOCS_DIR/discovery/prd.md`
- `$DOCS_DIR/discovery/ux-direction.md`

## Conditional Inputs (consumed by ux-designer sub-agent)

- `$ORG_ROOT/.jarfis-org/wiki/DESIGN/token-map.json` — if org registered and reusing existing tokens (text mode baseline; also referenced by FE later)
- `$ORG_ROOT/.jarfis-org/wiki/DESIGN/pages/{project}/` — if importing existing mockups
- `state.design.figmaPages[]` — figma mode only
- (Figma YAML, assets, section maps) — generated by the sub-agent itself in figma mode; not pre-existing inputs

---

## Common-component handling (handshake with main session)

Figma common components (header/footer/GNB) must be identified BEFORE Phase 3 spawns — the main session asks the user via AskUserQuestion and writes `state.design.common_components_skip`. jarfis-foreman in tmux cannot perform AskUserQuestion.

Contract:
- `state.design.common_components_skip` SHOULD be set by the main session before launching this phase.
- If unset (empty or missing), the ux-designer sub-agent applies a name-based heuristic only (nodes whose names contain `Header | Footer | GNB | Navigation`) with NO user confirmation, and records the heuristic application in phase-results meta.
- v4.1 may add a mid-phase pause handshake; v4.0 accepts heuristic-only as acceptable degradation.

jarfis-foreman passes `$COMMON_SKIP_JSON` to the ux-designer task prompt below.

## Step 1 — Figma reproduction (ux-designer, work-wide, 1 spawn)

### Compose invocation (jarfis-foreman executes)

```bash
COMPOSE=$(python3 ~/.claude/scripts/jarfis_cli.py compose \
  --agent ux-designer --state $STATE_FILE)
# Task spawn with the prompt below.
```

### Sub-agent task prompt (jarfis-foreman injects verbatim, with substitutions)

```
You are the UX designer reproducing Figma designs as pixel-perfect HTML page by page.
Execute the full pipeline PER PAGE in parallel (up to 5 concurrent pages).

Orchestrator-provided:
  figma_pages:       <figma_pages_json>         (array of {title, url})
  common_skip:       <common_skip_json>         (array of component names to skip)
  responsive:        <responsive>                (pc-only | pc-mobile | pc-mobile-tablet)

For each page in figma_pages:
  page_path = kebab-case of page.title   (e.g., "Benefits Signup" → "benefits-signup")
  Parse fileKey + nodeId from page.url

═══════════════════════════════════════════════
Per-page pipeline (steps F0 → F3c)
═══════════════════════════════════════════════

F0. Figma data extraction
  1. Call mcp__framelink__get_figma_data(fileKey, nodeId)
     Save YAML to $DOCS_DIR/design/{page_path}/figma-spec.yaml
  2. Call mcp__framelink__download_figma_images (scale=2)
     Save to $DOCS_DIR/design/{page_path}/reference.png
     Fallback on Framelink failure: Figma REST API
       GET /v1/images/{fileKey}?ids={nodeId}&format=png&scale=2

F1. Asset detection + download
  1. Parse the styles section of figma-spec.yaml — find `type: IMAGE` fills.
  2. Extract imageRef values; map to referencing node IDs + names.
  3. Write $DOCS_DIR/design/asset-manifest.json (aggregated across pages):
     [
       {"nodeId": "39286:42489", "imageRef": "abc123...",
        "fileName": "card-3d.png", "dimensions": {"width": 310, "height": 176}}
     ]
  4. Download: Framelink download_figma_images (primary) → Figma REST API (fallback, scale=2).
     Save to $DOCS_DIR/design/assets/{fileName}.
  5. Record verification at $DOCS_DIR/design/asset-verification.md:
     | fileName | nodeId | status | size |

F2. Token map generation (aggregated across pages — write ONCE per phase)
  1. Extract hex colors (solid + gradient stops) + typography from every figma-spec.yaml.
  2. Load design-system variables from each scope[].path/.jarfis-project/project-profile.md
     (Design System / Styles section). Parse variable files referenced.
  3. Match hex → variables:
     - Priority: semantic > theme > primitive.
     - Exact match → "exact": true.
     - Otherwise delta-E ≤ 5 fuzzy match → "exact": false, "deltaE": N.
     - Complete miss → record under "unmapped".
  4. Write $DOCS_DIR/design/token-map.json:
     {
       "colors": {
         "#1D77FF": {"scss":"$blue-600","css":"var(--brand-primary-600)",
                     "semantic":"var(--text-brand-primary)","exact":true},
         "#702818": {"scss":"$red-900","css":"var(--primitive-red-900)",
                     "semantic":null,"exact":false,"deltaE":3.2}
       },
       "typography": {"styles": {"Pretendard/18/700":{"fontSize":{...},...}}},
       "unmapped":   {"colors":["#AABBCC"],"note":"..."}
     }

F2.5. Section split (per page)
  1. Direct children of the YAML root frame = "sections".
  2. EXCLUDE nodes whose names match common_skip (apply name-based heuristic if common_skip
     is empty — match Header | Footer | GNB | Navigation).
  3. Merge height ≤ 40px adjacent nodes (dividers/spacers) into the previous section.
  4. Crop reference.png per section (y + height) via Playwright or an image tool;
     save to $DOCS_DIR/design/{page_path}/sections/{section-id}/reference.png.
  5. Write $DOCS_DIR/design/{page_path}/section-map.json:
     {
       "root": {"width":1440,"height":6294},
       "sections":[
         {"id":"section-0","name":"hero","nodeIds":["39286:42348"],"y":0,"height":480}
       ]
     }

F3a. Page shell generation (per page)
  Write $DOCS_DIR/design/{page_path}/index.html:
  - <!DOCTYPE html>, <head> (Pretendard CDN + basic reset)
  - Root container: apply YAML root width + background-color
  - For each section in section-map.json, insert a slot:
      <section id='section-{N}' class='section-slot'></section>
  - For common_skip items, insert placeholder:
      <!-- COMMON: {name} --> (reserve height)

F3b. Section-by-section generation + immediate verification (per section, iterative)
  For each section S in the page:
    a. Reproduce S as pixel-perfect HTML+CSS using the absolute forbidden rules
       and YAML → CSS mapping below.
    b. Save content.html at $DOCS_DIR/design/{page_path}/sections/{S.id}/content.html.
    c. Insert content.html into the page shell at {S.id}.
    d. Playwright screenshot (fullPage=true) of the page.
    e. Crop to section S's y/height area.
    f. Compare with sections/{S.id}/reference.png via mcp__design-comparison__compare_design.
    g. If differences exist, fix only that section (max 5 self-iterations per section).
    h. On PASS, move to next section.

F3c. Asset population + full assembly (per page)
  After ALL sections in a page PASS:
  1. Replace placeholders with real assets
     (photo → <img>, icon → <img>, bg → background-image).
  2. Assemble section content.html files into the final page index.html.
  3. Finalize $DOCS_DIR/design/{page_path}/index.html.

═══════════════════════════════════════════════
After all pages complete F3c: ToC + responsive capture
═══════════════════════════════════════════════

ToC:
  Regenerate $DOCS_DIR/design/_index.html linking to each page.

Responsive capture (per responsive value):
  pc-only          → keep the existing reference.png (1920×1080) per page; no extra capture
  pc-mobile        → also capture $DOCS_DIR/design/{page_path}/reference-mobile.png  (390×844)
  pc-mobile-tablet → also capture reference-mobile.png + reference-tablet.png (768×1024)
  Use mcp__playwright__browser_resize + mcp__playwright__browser_take_screenshot (fullPage=true).

═══════════════════════════════════════════════
Absolute forbidden rules (figma mode)
═══════════════════════════════════════════════
1. NO agent inference — do not infer styles not present in the spec.
   Example: "header should be dark because page is dark" is FORBIDDEN.
2. Strict node order — preserve sibling order exactly. Same-name nodes render twice; no dedup.
3. Image assets REQUIRED — IMAGE / IMAGE-SVG nodes must reference files under assets/.
   If an asset is missing, use `[MISSING_ASSET: {nodeId}]` placeholder.
4. Exact compound fill conversion — gradients + opacity converted to CSS precisely.
   No simplification / approximation.
5. Actively use token-map.json — prefer CSS variables over raw hex values.
6. 1:1 YAML → CSS mapping — follow the mapping guide below verbatim.
7. Vectors / icons → placeholders (.placeholder-icon, light bg #E1E4EC / dark bg #3D3E42).
   No SVG approximation.
8. Compound fills → .placeholder-bg (use first solid color). No multi-fill CSS reproduction.

═══════════════════════════════════════════════
YAML → CSS mapping guide
═══════════════════════════════════════════════
Layout:
  mode: row            → display:flex; flex-direction:row
  mode: column         → display:flex; flex-direction:column
  gap                  → gap
  padding              → padding
  justifyContent       → justify-content
  alignItems           → align-items
  sizing.horizontal: fill  → width:100%    (flex:1 in flex context)
  sizing.horizontal: fixed → width:{dimensions.width}px
  sizing.horizontal: hug   → width:fit-content
  clipsContent:true    → overflow:hidden
  borderRadius         → border-radius

Text:
  fontFamily / fontSize / fontWeight → font-family / font-size / font-weight
  lineHeight           → line-height
  letterSpacing        → letter-spacing
  textAlign            → text-align

Effects:
  dropShadow           → box-shadow
  innerShadow          → box-shadow: inset ...
  backgroundBlur       → backdrop-filter: blur(...)

Borders:
  stroke + strokeWeight → border
  strokeAlign: inside  → border (box-sizing:border-box)
  strokeAlign: outside → outline OR box-shadow

Gradients:
  linearGradient       → background: linear-gradient(...)
  radialGradient       → background: radial-gradient(...)

═══════════════════════════════════════════════
Report format when done
═══════════════════════════════════════════════
Per page emit:
  [FIGMA_PAGE_DONE: page_path={page_path} sections={N}]

At the very end, emit a summary:
  [FIGMA_ALL_DONE: pages={M} heuristic_common_skip_applied={true|false}]
```

jarfis-foreman fills `<figma_pages_json>`, `<common_skip_json>`, `<responsive>` before injection.

## Completion Protocol

At phase completion, perform the following in order:

1. **Produce spec artifacts**:
   - `$DOCS_DIR/design/{page_path}/index.html` (every page)
   - `$DOCS_DIR/design/{page_path}/reference.png` (+ `reference-mobile.png`, `reference-tablet.png` per responsive)
   - `$DOCS_DIR/design/_index.html` (ToC)
   - Additional for figma mode: `figma-spec.yaml` per page, `asset-manifest.json`, `assets/`, `token-map.json`, `section-map.json` per page, `sections/{id}/*` per page
   - Additional for text mode: `token-map.json`
   - Temp files remain at `$DOCS_DIR/design/tmp/round-{N}/*.md` for debugging
2. **(Optional) Handoff document**
3. **Write phase-results/phase3/attempt{K}.json** (last step):
   ```bash
   mkdir -p $DOCS_DIR/phase-results/phase3
   # Success
   cat > $DOCS_DIR/phase-results/phase3/attempt{K}.json <<EOF
   {
     "status": "completed",
     "reason": "",
     "reasonDetail": "",
     "meta": {
       "mode": "$DESIGN_MODE",
       "pages": <M>,
       "heuristic_common_skip_applied": <true|false>
     }
   }
   EOF
   # Error: status=error with reason/reasonDetail
   ```

**Strict order**: artifacts → (handoff) → phase-results JSON.
Do NOT write to `.jarfis-state.json` (the main Claude owns it).
`{K}` is substituted with the actual attempt number by the main session when generating the prompt file.
